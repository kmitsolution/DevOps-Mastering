# 🚨 1. Python Interpreter Warning (How to Suppress)

### ❗ Warning you see:

```text
Host is using discovered Python interpreter at '/usr/bin/python3.12'
```

👉 This is **NOT an error**, just a warning that:

* Ansible auto-detected Python
* Future changes might break detection

---

## ✅ Best Fix (Recommended)

Add this in your **inventory file**:

```ini
[webservers]
192.168.0.9 ansible_user=vboxuser ansible_python_interpreter=/usr/bin/python3
192.168.0.10 ansible_user=vboxuser ansible_python_interpreter=/usr/bin/python3
```

---

## ✅ OR Global Fix (Cleaner)

In `ansible.cfg`:

```ini
[defaults]
interpreter_python = auto_silent
```

👉 This will:

* Auto-detect Python
* ❌ Suppress warning

# ⚙️ 2. What is `become` in Ansible?

👉 **`become` = privilege escalation (like sudo)**

---

## 🔹 Without become

```bash
ansible webservers -m apt ...
```

👉 Runs as:

```bash
vboxuser
```

---

## 🔹 With become

```bash
ansible webservers -m apt ... --become
```

👉 Runs as:

```bash
root
```

---

## 🧠 Simple Explanation

| Concept  | Meaning               |
| -------- | --------------------- |
| SSH user | login user (vboxuser) |
| become   | switch to root (sudo) |

---

## 🔧 Internally

Ansible does:

```bash
ssh vboxuser@host → sudo → run command as root
```

---

# ⚙️ 3. How to Add `become` in ansible.cfg

Instead of writing `--become` every time 👇

---

## 🔹 Edit config file

```bash
sudo nano /etc/ansible/ansible.cfg
```

---

## 🔹 Add:

```ini
[defaults]
inventory = /etc/ansible/hosts
interpreter_python = auto_silent

[privilege_escalation]
become = True
become_method = sudo
become_user = root
become_ask_pass = False
```

---

## 🔍 Explanation

| Setting                 | Meaning            |
| ----------------------- | ------------------ |
| become = True           | Always use sudo    |
| become_method = sudo    | Use sudo           |
| become_user = root      | Switch to root     |
| become_ask_pass = False | No password prompt |
