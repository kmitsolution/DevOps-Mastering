# Running the React Application Locally

Before creating a CI/CD pipeline, it is important to verify that the application runs successfully on your local machine. This helps ensure that the source code is working correctly before automating the build process with GitLab CI.

In this section, we will install the required software, build the application, and run it locally.

---

# Prerequisites

Install the following software on your system:

* Visual Studio Code (VS Code)
* Node.js (includes npm)

After installing Node.js, both the **Node.js runtime** and the **Node Package Manager (npm)** are available.

---

# Step 1: Verify Node.js Installation

Open a terminal (or Command Prompt) and run:

```bash
node -v
```

Example Output:

```text
v22.18.0
```

### Explanation

* `node` is the JavaScript runtime used to execute JavaScript outside the browser.
* `-v` displays the installed Node.js version.
* If the version is displayed, Node.js has been installed successfully.

---

# Step 2: Verify npm Installation

Run:

```bash
npm -v
```

Example Output:

```text
10.9.3
```

### Explanation

* `npm` stands for **Node Package Manager**.
* It is used to install project dependencies, third-party libraries, and development tools.
* npm is installed automatically when you install Node.js.

---

# Step 3: Install Project Dependencies

Navigate to the project directory and execute:

```bash
npm install
```

### What does `npm install` do?

When you download or clone a Node.js project, you will notice that it contains a file called:

```text
package.json
```

The `package.json` file contains information about the project, including:

* Project name
* Version
* Scripts
* Required dependencies
* Development dependencies

Example:

```text
Project
│
├── package.json
├── src/
├── public/
└── README.md
```

When you execute:

```bash
npm install
```

npm performs the following tasks:

1. Reads the `package.json` file.
2. Downloads all required packages from the npm registry.
3. Installs the packages into the **node_modules** directory.
4. Creates or updates the **package-lock.json** file.

After installation, the project structure becomes:

```text
Project
│
├── node_modules/
├── src/
├── public/
├── package.json
└── package-lock.json
```

### What is `node_modules`?

The **node_modules** folder contains all the libraries and packages required by the application.

Examples:

* React
* React DOM
* Webpack
* Babel
* Other third-party packages

This folder is automatically generated by `npm install`.

> **Note:** The `node_modules` folder is usually **not committed** to Git because it can be recreated at any time using `npm install`.

---

# Step 4: Understanding the `src` Folder

The `src` folder contains the application's source code.

Typical structure:

```text
src/
│
├── App.js
├── index.js
├── App.css
├── components/
└── assets/
```

This is where developers write the React application.

When building the project, the files inside the `src` folder are compiled and optimized.

---

# Step 5: Build the React Application

Run:

```bash
npm run build
```

### What does this command do?

The command executes the **build** script defined in the `package.json` file.

Example:

```json
"scripts": {
  "build": "react-scripts build"
}
```

The build process:

* Compiles the React application.
* Bundles JavaScript and CSS files.
* Optimizes the application for production.
* Minifies the code.
* Creates static files.

After successful execution, a new folder named **build** is created.

Project structure:

```text
Project
│
├── build/
├── src/
├── public/
├── node_modules/
└── package.json
```

### What is the `build` folder?

The **build** folder contains the production-ready version of the application.

Typical contents:

```text
build/
│
├── index.html
├── static/
│   ├── css/
│   ├── js/
│   └── media/
└── asset-manifest.json
```

These files are deployed to a web server in production.

---

# Step 6: Install the Serve Package

Run:

```bash
npm install -g serve
```

### Explanation

* `serve` is a lightweight web server used to host static websites.
* `-g` stands for **global**.
* Installing globally makes the `serve` command available from any terminal on your computer.

---

# Step 7: Run the Application

Execute:

```bash
serve -s build/
```

### Explanation

* `serve` starts a local web server.
* `-s` means **serve in single-page application (SPA) mode**.
* `build/` specifies the folder containing the production build.

Example Output:

```text
Serving!

Local:    http://localhost:3000
Network:  http://192.168.1.5:3000
```

Open your browser and navigate to:

```text
http://localhost:3000
```

The React application will be displayed.

# Commands Summary

| Command                | Purpose                                                                                            |
| ---------------------- | -------------------------------------------------------------------------------------------------- |
| `node -v`              | Verify that Node.js is installed and display its version.                                          |
| `npm -v`               | Verify that npm is installed and display its version.                                              |
| `npm install`          | Install all project dependencies listed in `package.json` and create the `node_modules` directory. |
| `npm run build`        | Build the React application and generate the production-ready `build` folder.                      |
| `npm install -g serve` | Install the `serve` package globally to host static files.                                         |
| `serve -s build/`      | Start a local web server and serve the production build of the React application.                  |

---

# Summary

To run a React application locally, first install **Node.js** (which includes **npm**) and verify the installation using `node -v` and `npm -v`. Next, execute `npm install` to download all dependencies defined in the `package.json` file and create the `node_modules` directory. The application's source code resides in the `src` folder. Running `npm run build` compiles and optimizes the application, producing a production-ready `build` folder. Finally, install the `serve` package globally using `npm install -g serve` and start a local web server with `serve -s build/`. You can then access the application through your web browser using the local URL displayed in the terminal.
